# Setup Guide: Cloud-Native Development Environment

## 1. Overview
This guide covers setting up your **VMS Risk System** project for cross-machine synchronization using **GitHub** (for source code) and **Google Drive via Rclone** (for encrypted history).

## 2. Prerequisites
- [x] Docker installed and running.
- [x] Rclone installed (`curl https://rclone.org/install.sh | sudo bash`).
- [x] Git installed.
- [x] GitHub account and Google Drive access.

## 3. GitHub Initialization (Logic)
Run these commands in your `vms-risk-system/` directory:
```bash
git init
git add .
git commit -m "Initialize project structure with Docker support"
git remote add origin <your-repo-url>
git branch -M main
git push -u origin main
```

## 4. Google Drive Configuration (Data)
1. Run: `rclone config`
2. Follow prompts:
   - **Name**: `remote`
   - **Storage**: `drive` (Google Drive)
   - **Scope**: `1` (Full access)
   - **Auto Config**: `y` (Follow browser auth)
3. Create the remote directory: `rclone mkdir remote:vms-vault`

## 5. First-Time Synchronization
To push your state to the cloud:

```bash
# Push Code
git push -u origin main

# Push Encrypted Vault
rclone sync ../vms-vault/cipher remote:vms-vault/cipher
```

## 6. Daily Workflow
On any machine:
1. `./setup_docker.sh` (Initial setup)
2. `./sync.sh` (Pulls latest code and vault)
3. `docker-compose up` (Start environment)
